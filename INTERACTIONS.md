# comunikit — Документация взаимодействий

> Платформа: **comunikit** — закрытый студенческий маркетплейс AITUC  
> Дизайн: «Digital Bazaar» — тёплый оранжевый, небесно-синий, плотная сетка карточек  
> Версия прототипа: 1.0

---

## 1. Авторизация и регистрация

### 1.1 Страница входа (`/login`)

**Поведение формы при валидации:**

Форма использует клиентскую валидацию перед отправкой. При нажатии «Войти» проверяются следующие условия:

| Поле | Правило | Сообщение об ошибке |
|------|---------|---------------------|
| Email | Обязательное, содержит `@` | «Введите email» / «Некорректный email» |
| Пароль | Обязательное, минимум 6 символов | «Введите пароль» / «Минимум 6 символов» |

При наличии ошибок поля подсвечиваются красной рамкой (`border-destructive`), и под каждым полем появляется текст ошибки. Кнопка «Войти» не блокируется заранее — валидация происходит при сабмите. Успешный вход показывает toast «Добро пожаловать в comunikit!» и перенаправляет на `/feed`.

**Ссылка «Забыл пароль?»** — в прототипе показывает toast «Функция восстановления пароля в разработке». В продакшне: переход на страницу `/reset-password` с отправкой письма на email.

### 1.2 Страница регистрации (`/register`) — Whitelist Flow

Регистрация разбита на **2 шага** с визуальным индикатором прогресса.

**Шаг 1 — Верификация Student ID:**

1. Пользователь вводит Student ID (5–8 цифр, только числа).
2. Нажимает «Проверить» — запрос к API whitelist (в прототипе: имитация задержки 800 мс).
3. Если ID **найден**: поле «Полное имя» и «Код группы» автозаполняются, иконка ✅ появляется справа от поля, кнопка «Продолжить» становится активной.
4. Если ID **не найден**: иконка ❌, сообщение «ID не найден в базе студентов AITUC», кнопка «Продолжить» остаётся заблокированной.

> Тестовые ID для прототипа: `12345`, `23456`, `34567`, `99999`

**Шаг 2 — Данные аккаунта:**

Поля Email, Пароль, Подтвердить пароль. Валидация при сабмите:
- Email: обязательное, содержит `@`
- Пароль: минимум 8 символов
- Подтверждение: должно совпадать с паролем

## 2. Главная лента (`/feed`)

### 2.1 Переключение табов

При клике на таб (Все / Продажа / Покупка / Услуги / Потеряно / Найдено) происходит **мгновенная фильтрация** списка объявлений без перезагрузки страницы. Активный таб выделяется фоном `bg-primary text-white`. Количество объявлений обновляется под табами.

### 2.2 Фильтры

Кнопка с иконкой фильтра открывает/закрывает панель фильтров с анимацией `ck-animate-in` (slide-up 180ms). Доступные фильтры:
- **Категория**: выпадающий список (Электроника, Учёба, Одежда и т.д.)
- **Макс. цена**: числовое поле в тенге

Кнопка «Сбросить» обнуляет оба фильтра. Фильтры применяются в реальном времени через `useMemo`.

### 2.3 Переключение вид (Grid / List)

Кнопка с иконкой переключает между сеткой (2 колонки на мобильном, 3 на десктопе) и списком (одна колонка с горизонтальными карточками).

### 2.4 FAB «+ Добавить»

Плавающая кнопка в правом нижнем углу (только на мобильном, `lg:hidden`). При нажатии переходит на `/create`. Имеет эффект тени `box-shadow: 0 4px 20px oklch(0.705 0.191 37.5 / 40%)` и scale-анимацию при hover/active.

## 3. Карточка объявления (компонент `ListingCard`)

**Состояния:**

| Состояние | Поведение |
|-----------|-----------|
| Default | Карточка с тенью `shadow-sm` |
| Hover | `translateY(-1px)`, увеличенная тень, изображение scale 1.05 |
| Active (tap) | scale 0.97 |
| Liked | Иконка сердца заполняется красным (`fill-red-500`) |

**Цветовая полоска** слева от карточки (`.ck-card-stripe`) кодирует тип объявления:
- Оранжевый → Продажа
- Голубой → Покупка
- Фиолетовый → Услуга
- Красный → Потеряно
- Зелёный → Найдено

## 4. Детальная страница объявления (`/listing/:id`)

### 4.1 Кнопка «Показать контакт»

По умолчанию кнопка показывает текст «Показать контакт» и скрывает реальные данные. При первом нажатии:
1. Состояние `showContact` переключается в `true`.
2. Текст кнопки меняется на реальный контакт (Telegram handle или маскированный номер).
3. Показывается toast «Контакт раскрыт».

Это защищает контактные данные от скрейпинга и мотивирует авторизованных пользователей к взаимодействию.

### 4.2 Галерея изображений

Свайп между фотографиями реализован через кнопки «‹» и «›». Индикаторы (точки) внизу показывают текущий индекс. Активная точка расширяется до ширины 16px. Максимум 5 фотографий.

### 4.3 Комментарии

- Форма добавления комментария доступна авторизованным пользователям.
- При отправке пустого комментария кнопка «Отправить» заблокирована (`disabled`).
- Новый комментарий добавляется в начало списка с анимацией `ck-animate-in`.

## 5. Форма создания объявления (`/create`)

### 5.1 Пошаговый мастер

Форма разбита на **3 шага** с визуальным прогрессом:

| Шаг | Содержание | Условие перехода |
|-----|-----------|-----------------|
| 1 | Выбор типа (Маркетплейс / Lost&Found) + подтип | Оба выбраны |
| 2 | Заголовок, категория, цена, описание, фото, место | Заголовок ≥5 символов, категория выбрана, описание ≥10 символов |
| 3 | Подтверждение и публикация | — |

### 5.2 Предпросмотр

Кнопка «Предпросмотр» на шаге 2 показывает/скрывает блок с превью объявления (как оно будет выглядеть в ленте). Данные обновляются в реальном времени по мере заполнения формы.

### 5.3 Загрузка изображений

Drag&drop зона (в прототипе — кнопка «Добавить»). При выборе файлов через `FileReader` создаются base64-превью. Максимум 5 фотографий — при превышении показывается toast «Максимум 5 фотографий». Каждое фото можно удалить кнопкой ✕.

### 5.4 Карта для Lost & Found

Клик на карту-плейсхолдер имитирует выбор точки и показывает toast «Место выбрано на карте». В продакшне: интеграция с Google Maps API для выбора точки кликом.

## 6. Карта кампуса (`/map`)

### 6.1 Маркеры

Красные маркеры (🔴) = потеряно, зелёные (🟢) = найдено. При клике на маркер появляется popup с:
- Типом (потеряно/найдено)
- Заголовком объявления
- Местом и временем
- Кнопкой «Открыть объявление» → переход на `/listing/:id`

При повторном клике на тот же маркер popup закрывается.

### 6.2 Фильтр маркеров

Кнопки «Все / Потеряно / Найдено» мгновенно скрывают/показывают соответствующие маркеры.

## 7. Профиль (`/profile`)

### 7.1 Вкладки

Переключение между «Мои объявления», «Избранное», «Отзывы», «Настройки» происходит без перезагрузки страницы. Активная вкладка выделяется `bg-primary text-white`.

### 7.2 Настройки — тёмная тема

Переключатель «Тёмная тема» в настройках профиля синхронизирован с глобальным `ThemeProvider`. Изменение применяется мгновенно ко всему приложению и сохраняется в `localStorage`.

---

## 9. Тёмная тема

Переключение через кнопку в хедере или в настройках профиля. CSS-переменные в `.dark {}` переопределяют все цвета: фон `#111118` (deep navy-black), текст тёплый белый, карточки `oklch(0.18)`. Все компоненты используют семантические CSS-переменные (`bg-background`, `text-foreground`, `bg-card` и т.д.), поэтому тема применяется мгновенно без перезагрузки.
